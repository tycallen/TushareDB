## 规模合理性评估

### 1. 统计显著性

#### 置信区间计算

对于二项分布（信号触发与否），在 95% 置信水平下：

```
置信区间 = p̂ ± 1.96 × √(p̂(1-p̂)/n)
```

其中：
- p̂ = 触发概率估计值
- n = 总样本数

#### 实际计算示例

假设触发概率 p̂ = 5% (0.05)：

| 路径数 | 天数 | 总样本 | 95% 置信区间半宽 | 相对误差 |
|--------|------|--------|------------------|----------|
| 1,000 | 252 | 252,000 | ±0.27% | ±5.4% |
| 5,000 | 252 | 1,260,000 | ±0.12% | ±2.4% |
| **10,000** | **252** | **2,520,000** | **±0.09%** | **±1.7%** |
| 50,000 | 252 | 12,600,000 | ±0.04% | ±0.8% |

**结论**：
- 10,000 路径的相对误差约 ±1.7%，对于因子筛选足够精确
- 5,000 路径的相对误差约 ±2.4%，可用于快速筛选
- 1,000 路径误差较大（±5.4%），仅适合演示

### 4. 统计功效分析

#### 检验力 (Power Analysis)

要检测 Alpha Ratio = 1.5 (即 P_actual = 1.5 × P_random) 的显著差异：

```python
# 使用两样本比例检验
from statsmodels.stats.power import zt_ind_solve_power

# 参数
effect_size = 0.05  # 5% 差异 (例如 5% vs 3.33%)
alpha = 0.05        # 显著性水平
power = 0.80        # 检验力 80%

# 计算所需样本
n = zt_ind_solve_power(effect_size=effect_size, alpha=alpha, power=power)
# 结果: 每组约 1,200 样本
```

**对于 P_random = 5% 的因子：**
- 需要检测 P_actual = 7.5% (Alpha=1.5)
- 10,000 路径 × 252 天 = 2,520,000 样本 >> 1,200
- **功效充足 (Power > 99%)**

### 5. 收敛性验证

运行不同规模，观察 Alpha Ratio 的稳定性：

```python
results = []
for n in [100, 500, 1000, 5000, 10000, 50000]:
    # 运行验证
    report = filter_obj.filter(factor, n_simulations=n)
```

**预期结果**：

- n = 5000：Alpha 基本稳定 (±5%)
- **n = 10000：Alpha 收敛 (±2%)**
- n > 10000：边际改善很小

## 当前规模评估结论

### ✅ 10,000 × 252 是合理的

**优点：**
1. **统计精度高**：置信区间 ±1.7%，满足筛选需求
2. **计算速度快**：单次 < 0.2 秒，可批量处理
3. **内存友好**：仅 20MB，不影响其他程序
4. **功效充足**：检测 Alpha=1.5 的功效 > 99%

**缺点：**
1. 对于 P_random < 1% 的因子，信号数较少 (~25,000)
2. 无法检测非常微弱的 Alpha (如 1.1)
